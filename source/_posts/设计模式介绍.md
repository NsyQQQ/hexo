设计模式是软件开发中经过验证的解决方案，可以帮助我们编写更可维护、更灵活、更优雅的代码。本文将介绍常见的设计模式，帮助你提升编程水平。

## 1. 设计模式简介

### 1.1 什么是设计模式

设计模式是在软件开发中反复出现的问题的可复用解决方案。它们不是具体的代码实现，而是一种经过验证的编程思想。

### 1.2 设计模式分类

设计模式主要分为三类：

- **创建型模式**：对象的创建机制
- **结构型模式**：对象组合成更大结构
- **行为型模式**：对象间的通信机制

## 2. 创建型模式

### 2.1 单例模式

确保一个类只有一个实例，并提供全局访问点。

```javascript
// 懒汉式
class Singleton {
    constructor() {
        if (Singleton.instance) {
            return Singleton.instance;
        }
        Singleton.instance = this;
        this.data = [];
    }
    
    getInstance() {
        if (!Singleton.instance) {
            Singleton.instance = new Singleton();
        }
        return Singleton.instance;
    }
}

// 饿汉式
class SingletonEager {
    static instance = new SingletonEager();
    
    constructor() {
        this.data = [];
    }
}

// 使用
const s1 = new Singleton();
const s2 = new Singleton();
console.log(s1 === s2); // true
```

### 2.2 工厂模式

封装对象的创建过程。

```javascript
// 简单工厂
class Car {
    constructor(type) {
        this.type = type;
    }
}

class CarFactory {
    createCar(type) {
        switch (type) {
            case 'sedan': return new Car('轿车');
            case 'suv': return new Car('SUV');
            case 'truck': return new Car('卡车');
            default: throw new Error('未知车型');
        }
    }
}

// 工厂方法
class Factory {
    create() {
        throw new Error('必须在子类中实现');
    }
}

class ConcreteFactoryA extends Factory {
    create() {
        return new ProductA();
    }
}

class ConcreteFactoryB extends Factory {
    create() {
        return new ProductB();
    }
}
```

### 2.3 抽象工厂模式

提供创建一系列相关对象的接口。

```javascript
class Button {
    render() { console.log('渲染按钮'); }
}

class Input {
    render() { console.log('渲染输入框'); }
}

// 主题工厂接口
class ThemeFactory {
    createButton() { throw new Error('必须实现'); }
    createInput() { throw new Error('必须实现'); }
}

// 亮色主题工厂
class LightThemeFactory extends ThemeFactory {
    createButton() {
        return { style: 'light', render: () => console.log('亮色按钮') };
    }
    createInput() {
        return { style: 'light', render: () => console.log('亮色输入框') };
    }
}

// 暗色主题工厂
class DarkThemeFactory extends ThemeFactory {
    createButton() {
        return { style: 'dark', render: () => console.log('暗色按钮') };
    }
    createInput() {
        return { style: 'dark', render: () => console.log('暗色输入框') };
    }
}
```

### 2.4 建造者模式

将对象的构建与表示分离。

```javascript
class Computer {
    constructor() {
        this.cpu = '';
        this.ram = '';
        this.storage = '';
    }
    
    toString() {
        return `CPU: ${this.cpu}, RAM: ${this.ram}, Storage: ${this.storage}`;
    }
}

class ComputerBuilder {
    constructor() {
        this.computer = new Computer();
    }
    
    setCPU(cpu) {
        this.computer.cpu = cpu;
        return this;
    }
    
    setRAM(ram) {
        this.computer.ram = ram;
        return this;
    }
    
    setStorage(storage) {
        this.computer.storage = storage;
        return this;
    }
    
    build() {
        return this.computer;
    }
}

// 使用
const computer = new ComputerBuilder()
    .setCPU('Intel i7')
    .setRAM('16GB')
    .setStorage('512GB SSD')
    .build();
```

### 2.5 原型模式

通过克隆现有对象创建新对象。

```javascript
class Prototype {
    constructor() {
        this.data = {};
    }
    
    clone() {
        const clone = new Prototype();
        clone.data = { ...this.data };
        return clone;
    }
}

const original = new Prototype();
original.data = { name: '原始对象', value: 100 };

const clone = original.clone();
clone.data.name = '克隆对象';

console.log(original.data.name); // 原始对象
console.log(clone.data.name);    // 克隆对象
```

## 3. 结构型模式

### 3.1 装饰器模式

动态地给对象添加额外职责。

```javascript
class Coffee {
    cost() { return 5; }
    description() { return '咖啡'; }
}

class MilkDecorator {
    constructor(coffee) {
        this.coffee = coffee;
    }
    
    cost() {
        return this.coffee.cost() + 1;
    }
    
    description() {
        return this.coffee.description() + ', 牛奶';
    }
}

class SugarDecorator {
    constructor(coffee) {
        this.coffee = coffee;
    }
    
    cost() {
        return this.coffee.cost() + 0.5;
    }
    
    description() {
        return this.coffee.description() + ', 糖';
    }
}

// 使用
let coffee = new Coffee();
coffee = new MilkDecorator(coffee);
coffee = new SugarDecorator(coffee);

console.log(coffee.description()); // 咖啡, 牛奶, 糖
console.log(coffee.cost());        // 6.5
```

### 3.2 适配器模式

将不兼容的接口转换为兼容的接口。

```javascript
// 旧接口
class OldPayment {
    pay(amount) {
        console.log(`旧接口支付: ${amount}元`);
    }
}

// 新接口
class NewPayment {
    makePayment(details) {
        console.log(`新接口支付: ${details.amount}元, 方式: ${details.method}`);
    }
}

// 适配器
class PaymentAdapter {
    constructor() {
        this.newPayment = new NewPayment();
    }
    
    pay(amount) {
        this.newPayment.makePayment({
            amount: amount,
            method: '支付宝'
        });
    }
}

// 使用
const adapter = new PaymentAdapter();
adapter.pay(100); // 新接口支付: 100元, 方式: 支付宝
```

### 3.3 代理模式

为对象提供一个替身以控制对它的访问。

```javascript
class RealImage {
    constructor(filename) {
        this.filename = filename;
        this.loadFromDisk();
    }
    
    loadFromDisk() {
        console.log(`加载图片: ${this.filename}`);
    }
    
    display() {
        console.log(`显示图片: ${this.filename}`);
    }
}

// 代理类
class ImageProxy {
    constructor(filename) {
        this.filename = filename;
        this.realImage = null;
    }
    
    display() {
        if (!this.realImage) {
            this.realImage = new RealImage(this.filename);
        }
        this.realImage.display();
    }
}

// 使用
const image = new ImageProxy('photo.jpg');
// 延迟加载：只有调用 display 才真正加载
image.display();
```

### 3.4 外观模式

提供统一的接口来访问子系统中的一组接口。

```javascript
class CPU {
    start() { console.log('CPU 启动'); }
    shutdown() { console.log('CPU 关闭'); }
}

class Memory {
    start() { console.log('内存 启动'); }
    shutdown() { console.log('内存 关闭'); }
}

class Disk {
    start() { console.log('硬盘 启动'); }
    shutdown() { console.log('硬盘 关闭'); }
}

// 外观类
class ComputerFacade {
    constructor() {
        this.cpu = new CPU();
        this.memory = new Memory();
        this.disk = new Disk();
    }
    
    start() {
        console.log('=== 开机 ===');
        this.cpu.start();
        this.memory.start();
        this.disk.start();
        console.log('=== 开机完成 ===');
    }
    
    shutdown() {
        console.log('=== 关机 ===');
        this.cpu.shutdown();
        this.memory.shutdown();
        this.disk.shutdown();
        console.log('=== 关机完成 ===');
    }
}

// 使用
const computer = new ComputerFacade();
computer.start();
computer.shutdown();
```

### 3.5 组合模式

将对象组合成树形结构以表示"部分-整体"的层次结构。

```javascript
class Component {
    constructor(name) {
        this.name = name;
    }
    
    add(component) { throw new Error('不支持操作'); }
    remove(component) { throw new Error('不支持操作'); }
    display(depth) { throw new Error('必须实现'); }
}

class Leaf extends Component {
    display(depth) {
        console.log('-'.repeat(depth) + this.name);
    }
}

class Composite extends Component {
    constructor(name) {
        super(name);
        this.children = [];
    }
    
    add(component) {
        this.children.push(component);
    }
    
    remove(component) {
        const index = this.children.indexOf(component);
        if (index > -1) this.children.splice(index, 1);
    }
    
    display(depth) {
        console.log('-'.repeat(depth) + this.name);
        for (const child of this.children) {
            child.display(depth + 2);
        }
    }
}

// 使用
const root = new Composite('root');
const branch1 = new Composite('branch1');
const leaf1 = new Leaf('leaf1');
const leaf2 = new Leaf('leaf2');

branch1.add(leaf1);
branch1.add(leaf2);
root.add(branch1);

root.display(0);
```

## 4. 行为型模式

### 4.1 观察者模式

定义对象间的一对多依赖关系。

```javascript
class Subject {
    constructor() {
        this.observers = [];
    }
    
    attach(observer) {
        this.observers.push(observer);
    }
    
    detach(observer) {
        const index = this.observers.indexOf(observer);
        if (index > -1) this.observers.splice(index, 1);
    }
    
    notify() {
        for (const observer of this.observers) {
            observer.update(this);
        }
    }
}

class ConcreteSubject extends Subject {
    constructor() {
        super();
        this.state = 0;
    }
    
    setState(state) {
        this.state = state;
        this.notify();
    }
    
    getState() {
        return this.state;
    }
}

class Observer {
    update(subject) {
        console.log(`观察者收到通知，状态: ${subject.getState()}`);
    }
}

// 使用
const subject = new ConcreteSubject();
const observer1 = new Observer();
const observer2 = new Observer();

subject.attach(observer1);
subject.attach(observer2);

subject.setState(5); // 观察者收到通知，状态: 5
```

### 4.2 策略模式

定义一系列算法，将每个算法封装起来。

```javascript
class PaymentStrategy {
    pay(amount) {
        throw new Error('必须实现');
    }
}

class AlipayStrategy extends PaymentStrategy {
    pay(amount) {
        console.log(`使用支付宝支付: ${amount}元`);
    }
}

class WechatPayStrategy extends PaymentStrategy {
    pay(amount) {
        console.log(`使用微信支付: ${amount}元`);
    }
}

class CreditCardStrategy extends PaymentStrategy {
    pay(amount) {
        console.log(`使用信用卡支付: ${amount}元`);
    }
}

class ShoppingCart {
    constructor() {
        this.items = [];
    }
    
    addItem(item, price) {
        this.items.push({ item, price });
    }
    
    setPaymentStrategy(strategy) {
        this.strategy = strategy;
    }
    
    checkout() {
        const total = this.items.reduce((sum, item) => sum + item.price, 0);
        this.strategy.pay(total);
    }
}

// 使用
const cart = new ShoppingCart();
cart.addItem('商品1', 100);
cart.addItem('商品2', 200);

cart.setPaymentStrategy(new AlipayStrategy());
cart.checkout(); // 使用支付宝支付: 300元
```

### 4.3 命令模式

将请求封装成对象。

```javascript
class Command {
    execute() { throw new Error('必须实现'); }
    undo() { throw new Error('必须实现'); }
}

class Light {
    on() { console.log('灯亮了'); }
    off() { console.log('灯关了'); }
}

class LightOnCommand extends Command {
    constructor(light) {
        super();
        this.light = light;
    }
    
    execute() { this.light.on(); }
    undo() { this.light.off(); }
}

class LightOffCommand extends Command {
    constructor(light) {
        super();
        this.light = light;
    }
    
    execute() { this.light.off(); }
    undo() { this.light.on(); }
}

class RemoteControl {
    constructor() {
        this.history = [];
    }
    
    executeCommand(command) {
        command.execute();
        this.history.push(command);
    }
    
    undoLast() {
        const command = this.history.pop();
        if (command) command.undo();
    }
}

// 使用
const light = new Light();
const lightOn = new LightOnCommand(light);
const lightOff = new LightOffCommand(light);

const remote = new RemoteControl();
remote.executeCommand(lightOn); // 灯亮了
remote.executeCommand(lightOff); // 灯关了
remote.undoLast(); // 灯亮了
```

### 4.4 迭代器模式

顺序访问集合元素而不暴露内部表示。

```javascript
class Iterator {
    constructor(collection) {
        this.collection = collection;
        this.position = 0;
    }
    
    hasNext() {
        return this.position < this.collection.length;
    }
    
    next() {
        if (this.hasNext()) {
            return this.collection[this.position++];
        }
        return null;
    }
    
    reset() {
        this.position = 0;
    }
}

// 使用
const collection = [1, 2, 3, 4, 5];
const iterator = new Iterator(collection);

while (iterator.hasNext()) {
    console.log(iterator.next());
}
```

### 4.5 模板方法模式

定义算法框架，将某些步骤延迟到子类。

```javascript
class DataMiner {
    // 模板方法
    mine(path) {
        const file = this.openFile(path);
        const rawData = this.extractData(file);
        const data = this.parseData(rawData);
        const analysis = this.analyzeData(data);
        this.sendReport(analysis);
        this.closeFile(file);
    }
    
    openFile(path) { throw new Error('必须实现'); }
    extractData(file) { throw new Error('必须实现'); }
    parseData(rawData) { throw new Error('必须实现'); }
    analyzeData(data) { console.log('分析数据:', data); return data; }
    sendReport(analysis) { console.log('发送报告'); }
    closeFile(file) { console.log('关闭文件'); }
}

class PDFDataMiner extends DataMiner {
    openFile(path) {
        console.log(`打开PDF: ${path}`);
        return { type: 'pdf', path };
    }
    
    extractData(file) {
        console.log('提取PDF数据');
        return 'PDF原始数据';
    }
    
    parseData(rawData) {
        console.log('解析PDF数据');
        return { format: 'pdf', content: rawData };
    }
}

// 使用
const miner = new PDFDataMiner();
miner.mine('report.pdf');
```

### 4.6 状态模式

允许对象在内部状态改变时改变行为。

```javascript
class State {
    insertCoin() { throw new Error('必须实现'); }
    ejectCoin() { throw new Error('必须实现'); }
    turnCrank() { throw new Error('必须实现'); }
    dispense() { throw new Error('必须实现'); }
}

class NoCoinState extends State {
    constructor(machine) {
        super();
        this.machine = machine;
    }
    
    insertCoin() {
        console.log('投币成功');
        this.machine.setState(this.machine.hasCoinState);
    }
    
    ejectCoin() {
        console.log('没有硬币可退回');
    }
    
    turnCrank() {
        console.log('请先投币');
    }
    
    dispense() {
        console.log('请先投币');
    }
}

class HasCoinState extends State {
    constructor(machine) {
        super();
        this.machine = machine;
    }
    
    insertCoin() {
        console.log('已经投币了');
    }
    
    ejectCoin() {
        console.log('退回硬币');
        this.machine.setState(this.machine.noCoinState);
    }
    
    turnCrank() {
        console.log('转动曲柄');
        this.machine.setState(this.machine.soldState);
    }
    
    dispense() {
        console.log('请先转动曲柄');
    }
}

class VendingMachine {
    constructor() {
        this.noCoinState = new NoCoinState(this);
        this.hasCoinState = new HasCoinState(this);
        this.soldState = new SoldState(this);
        this.state = this.noCoinState;
    }
    
    setState(state) {
        this.state = state;
    }
    
    insertCoin() { this.state.insertCoin(); }
    ejectCoin() { this.state.ejectCoin(); }
    turnCrank() { this.state.turnCrank(); }
    dispense() { this.state.dispense(); }
}
```

### 4.7 职责链模式

将请求的发送者和接收者解耦。

```javascript
class Handler {
    setNext(handler) {
        this.nextHandler = handler;
        return handler;
    }
    
    handle(request) {
        if (this.nextHandler) {
            return this.nextHandler.handle(request);
        }
        return null;
    }
}

class AuthHandler extends Handler {
    handle(request) {
        if (!request.isAuthenticated) {
            console.log('认证失败');
            return '认证失败';
        }
        console.log('认证通过');
        return super.handle(request);
    }
}

class ValidationHandler extends Handler {
    handle(request) {
        if (!request.data) {
            console.log('数据验证失败');
            return '数据验证失败';
        }
        console.log('数据验证通过');
        return super.handle(request);
    }
}

class ProcessHandler extends Handler {
    handle(request) {
        console.log('处理请求');
        return '处理完成';
    }
}

// 使用
const auth = new AuthHandler();
const validation = new ValidationHandler();
const process = new ProcessHandler();

auth.setNext(validation).setNext(process);

auth.handle({ isAuthenticated: true, data: { name: 'test' } });
```

## 5. 总结

本文介绍了常见的设计模式：

### 创建型模式
- **单例模式**：确保唯一实例
- **工厂模式**：封装对象创建
- **抽象工厂**：创建系列对象
- **建造者模式**：构建复杂对象
- **原型模式**：克隆对象

### 结构型模式
- **装饰器模式**：动态添加功能
- **适配器模式**：接口转换
- **代理模式**：访问控制
- **外观模式**：统一接口
- **组合模式**：树形结构

### 行为型模式
- **观察者模式**：一对多依赖
- **策略模式**：算法封装
- **命令模式**：请求封装
- **迭代器模式**：顺序访问
- **模板方法模式**：算法框架
- **状态模式**：状态改变行为
- **职责链模式**：链式处理

掌握这些设计模式，能够帮助你编写更高质量的代码，提升软件设计能力。
