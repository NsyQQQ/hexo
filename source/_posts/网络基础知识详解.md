## 一、网络协议概述

网络协议是计算机网络中进行数据交换而建立的规则、标准或约定的集合。它规定了通信双方如何建立连接、如何传输数据以及如何处理错误等。

### 常见的网络协议

| 协议 | 层次 | 说明 |
|------|------|------|
| HTTP | 应用层 | Web 数据传输 |
| DNS | 应用层 | 域名解析 |
| TCP | 传输层 | 可靠传输 |
| IP | 网络层 | 路由转发 |
| Ethernet | 数据链路层 | 局域网传输 |

## 二、TCP/IP 协议栈

TCP/IP 是互联网的基础协议族，它是一组协议的集合，包括 TCP、IP、UDP、HTTP 等多个协议。

### TCP/IP 四层模型

```
┌─────────────────────────────┐
│       应用层 (Application)    │  HTTP, DNS, FTP, SMTP
├─────────────────────────────┤
│       传输层 (Transport)      │  TCP, UDP
├─────────────────────────────┤
│       网络层 (Internet)       │  IP, ICMP, ARP
├─────────────────────────────┤
│     网络接口层 (Link)        │  Ethernet, Wi-Fi
└─────────────────────────────┘
```

### 各层功能说明

**应用层**
- 为用户提供网络应用服务
- 直接与用户交互
- 协议：HTTP、HTTPS、FTP、DNS、SMTP 等

**传输层**
- 提供端到端的可靠传输
- 负责流量控制、差错控制
- 协议：TCP（可靠）、UDP（快速）

**网络层**
- 实现主机到主机的通信
- 负责路由和寻址
- 协议：IP、ICMP、ARP

**网络接口层**
- 负责物理设备之间的数据传输
- 协议：Ethernet、PPP

## 三、IP 地址

IP 地址是分配给网络设备的唯一标识符，用于在网络中定位设备。

### IPv4 地址

IPv4 地址由 32 位二进制组成，通常用点分十进制表示：

```
192.168.1.1
```

IPv4 地址分类：

| 类别 | 范围 | 用途 |
|------|------|------|
| A 类 | 1.0.0.0 - 126.255.255.255 | 大型网络 |
| B 类 | 128.0.0.0 - 191.255.255.255 | 中型网络 |
| C 类 | 192.0.0.0 - 223.255.255.255 | 小型网络 |
| D 类 | 224.0.0.0 - 239.255.255.255 | 多播 |
| E 类 | 240.0.0.0 - 255.255.255.255 | 保留 |

### 特殊 IP 地址

- **127.0.0.1**：本地回环地址
- **0.0.0.0**：代表所有网络接口
- **255.255.255.255**：受限广播地址
- **192.168.x.x**：私有地址（局域网）
- **10.x.x.x**：私有地址（局域网）
- **172.16.x.x - 172.31.x.x**：私有地址（局域网）

### IPv6 地址

IPv6 地址由 128 位组成，用冒号十六进制表示：

```
2001:0db8:85a3:0000:0000:8a2e:0370:7334
```

简化表示：
```
2001:db8:85a3::8a2e:370:7334
```

### 子网掩码

子网掩码用于区分 IP 地址中的网络部分和主机部分：

```
IP 地址:    192.168.1.100
子网掩码:   255.255.255.0
网络地址:   192.168.1.0
主机地址:   100
```

常用子网掩码：
- /24 = 255.255.255.0（254 个主机）
- /16 = 255.255.0.0（65534 个主机）
- /8 = 255.0.0.0（16777214 个主机）

## 四、TCP 协议

TCP（Transmission Control Protocol）是面向连接的可靠传输协议。

### TCP 特点

- **面向连接**：建立连接后传输数据
- **可靠传输**：确认机制、重传机制
- **全双工**：可同时收发数据
- **字节流**：无消息边界

### TCP 三次握手

建立 TCP 连接需要三次握手：

```
客户端                              服务器
  │                                   │
  │───── SYN = 1, seq = x ──────────▶│
  │                                   │
  │◀──── SYN = 1, ACK = 1,           │
  │        seq = y, ack = x + 1 ─────│
  │                                   │
  │───── ACK = 1, seq = x + 1,      │
  │        ack = y + 1 ──────────────▶│
  │                                   │
  │          连接建立完成              │
```

**三次握手过程：**

1. **第一次握手**：客户端发送 SYN 包（SYN=1，seq=x），进入 SYN_SENT 状态
2. **第二次握手**：服务器收到 SYN 包，发送 SYN+ACK 包（SYN=1，ACK=1，seq=y，ack=x+1），进入 SYN_RCVD 状态
3. **第三次握手**：客户端收到 SYN+ACK 包，发送 ACK 包（ACK=1，seq=x+1，ack=y+1），进入 ESTABLISHED 状态

### TCP 四次挥手

关闭 TCP 连接需要四次挥手：

```
客户端                              服务器
  │                                   │
  │───── FIN = 1, seq = u ──────────▶│
  │◀──── ACK = 1, ack = u + 1 ───────│
  │          服务器继续发送数据          │
  │◀──── FIN = 1, ACK = 1,           │
  │        seq = w, ack = u + 1 ─────│
  │                                   │
  │───── ACK = 1, ack = w + 1 ──────▶│
  │          连接关闭完成              │
```

### TCP 状态转换

常见状态：
- LISTEN：监听连接
- SYN_SENT：已发送 SYN
- SYN_RCVD：已收到 SYN
- ESTABLISHED：连接已建立
- FIN_WAIT_1：已发送 FIN
- FIN_WAIT_2：已收到对方 ACK
- CLOSING：双方同时关闭
- TIME_WAIT：等待最后确认
- CLOSED：连接已关闭

### TCP 窗口与流量控制

TCP 使用滑动窗口机制进行流量控制：

```
发送窗口                    接收窗口
┌─────┬───────────────┬─────┐
│已确认│   可发送区域   │不可发送│
│ 100  │   101-200    │ 201+ │
└─────┴───────────────┴─────┘
```

- **拥塞控制**：防止网络过载
- **慢启动**：从小窗口开始
- **拥塞避免**：线性增长
- **快速重传**：收到 3 个重复 ACK

## 五、UDP 协议

UDP（User Datagram Protocol）是面向无连接的不可靠传输协议。

### UDP 特点

- **无连接**：不需要建立连接
- **不可靠**：不保证送达
- **高效**：头部开销小（8 字节）
- **无流量控制**：适合实时应用

### UDP 应用场景

- DNS 查询
- 视频流
- 语音通话
- 在线游戏
- DHCP

### TCP vs UDP

| 特性 | TCP | UDP |
|------|-----|-----|
| 连接性 | 面向连接 | 无连接 |
| 可靠性 | 可靠 | 不可靠 |
| 头部 | 20-60 字节 | 8 字节 |
| 速度 | 较慢 | 较快 |
| 适用 | 文件、邮件 | 实时、音视频 |

## 六、DNS 域名系统

DNS（Domain Name System）是互联网的域名解析系统。

### 域名结构

```
www.example.com
│  │      │      │
│  │      │      └── 顶级域名 TLD
│  │      └──────────── 二级域名 SLN
│  └─────────────────── 三级域名（主机名）
└─────────────────────── 根域名（隐藏）
```

### 域名级别

- **根域名**：. （点）
- **顶级域名**：.com、.org、.edu、.cn、.net 等
- **二级域名**：example、google、baidu 等
- **子域名**：www、mail、blog 等

### DNS 查询流程

```
1. 客户端查询 www.example.com
           │
           ▼
2. 查询本地 DNS 缓存
           │
           ▼ (未命中)
3. 查询根域名服务器 (.)
           │
           ▼ 返回 .com 服务器
4. 查询 .com 域名服务器
           │
           ▼ 返回 example.com DNS
5. 查询 example.com DNS
           │
           ▼ 返回 IP 地址
6. 返回结果给客户端
```

### DNS 记录类型

| 类型 | 说明 | 例子 |
|------|------|------|
| A | IPv4 地址 | example.com -> 93.184.216.34 |
| AAAA | IPv6 地址 | example.com -> 2606:2800:220:1 |
| CNAME | 别名 | www -> @ |
| MX | 邮件服务器 | @ -> mail.example.com |
| TXT | 文本记录 | SPF、验证信息 |
| NS | 域名服务器 | example.com -> ns1.example.com |
| SOA | 授权起始 | 域名管理信息 |

### 常用 DNS 命令

```bash
# 查询域名 IP
nslookup example.com
dig example.com
host example.com

# 指定 DNS 服务器查询
nslookup example.com 8.8.8.8
dig @8.8.8.8 example.com

# 查询 DNS 记录
dig example.com MX
dig example.com TXT
```

## 七、HTTP 协议

HTTP（HyperText Transfer Protocol）是 Web 应用的基础协议。

### HTTP 请求结构

```
请求行: GET /index.html HTTP/1.1
请求头: Host: example.com
       User-Agent: Mozilla/5.0
       Accept: text/html
       ...
空行:
请求体: (可选) POST 数据
```

### HTTP 响应结构

```
状态行: HTTP/1.1 200 OK
响应头: Content-Type: text/html
        Content-Length: 1234
        Server: Nginx
        ...
空行:
响应体: <html>...</html>
```

### HTTP 方法

| 方法 | 说明 | 幂等 |
|------|------|------|
| GET | 获取资源 | 是 |
| POST | 提交数据 | 否 |
| PUT | 上传资源 | 是 |
| DELETE | 删除资源 | 是 |
| PATCH | 部分更新 | 否 |
| HEAD | 获取响应头 | 是 |
| OPTIONS | 获取支持的方法 | 是 |

### HTTP 状态码

**1xx - 信息响应**
- 100 Continue
- 101 Switching Protocols

**2xx - 成功响应**
- 200 OK
- 201 Created
- 204 No Content

**3xx - 重定向**
- 301 Moved Permanently
- 302 Found
- 304 Not Modified

**4xx - 客户端错误**
- 400 Bad Request
- 401 Unauthorized
- 403 Forbidden
- 404 Not Found
- 405 Method Not Allowed

**5xx - 服务器错误**
- 500 Internal Server Error
- 502 Bad Gateway
- 503 Service Unavailable
- 504 Gateway Timeout

### HTTP 请求头

| 请求头 | 说明 |
|--------|------|
| Host | 服务器域名 |
| User-Agent | 客户端信息 |
| Accept | 可接受的 MIME 类型 |
| Accept-Language | 可接受的语言 |
| Accept-Encoding | 可接受的编码 |
| Cookie | Cookie 信息 |
| Referer | 请求来源 |
| Authorization | 认证信息 |

### HTTP 响应头

| 响应头 | 说明 |
|--------|------|
| Content-Type | 响应类型 |
| Content-Length | 响应长度 |
| Content-Encoding | 编码方式 |
| Server | 服务器信息 |
| Set-Cookie | 设置 Cookie |
| Cache-Control | 缓存控制 |
| Expires | 过期时间 |
| Last-Modified | 最后修改 |
| ETag | 资源标识 |

### HTTP 版本

**HTTP/1.0**
- 每次请求建立新连接
- 不支持持久连接
- 支持少量方法

**HTTP/1.1**
- 支持持久连接
- 支持管道化
- 支持分块传输
- 增加更多方法

**HTTP/2**
- 多路复用（并行传输）
- Header 压缩
- 服务器推送
- 二进制分帧

**HTTP/3**
- 基于 QUIC 协议
- 0-RTT 连接
- 更好的移动端体验
- 解决队头阻塞

### HTTPS

HTTPS = HTTP + TLS/SSL，提供加密传输。

```
┌─────────────────────────────────────────┐
│              HTTPS 流程                   │
├─────────────────────────────────────────┤
│ 1. 客户端发起 HTTPS 请求                │
│ 2. 服务器返回证书（公钥）               │
│ 3. 客户端验证证书有效性                 │
│ 4. 生成随机密钥，用公钥加密              │
│ 5. 服务器用私钥解密，获取密钥            │
│ 6. 双方使用对称密钥加密通信              │
└─────────────────────────────────────────┘
```

## 八、常见网络命令

### ping 命令

```bash
# 测试连通性
ping example.com

# 指定次数
ping -c 4 example.com

# 指定数据包大小
ping -s 1000 example.com
```

### curl 命令

```bash
# 发送 GET 请求
curl https://example.com

# 发送 POST 请求
curl -X POST -d "name=value" https://example.com

# 显示响应头
curl -I https://example.com

# 保存响应
curl -o filename.html https://example.com
```

### netstat 命令

```bash
# 查看所有连接
netstat -an

# 查看监听端口
netstat -tuln

# 查看进程
netstat -p
```

### telnet 命令

```bash
# 测试端口连通性
telnet example.com 80

# 发送 HTTP 请求
telnet example.com 80
GET / HTTP/1.1
Host: example.com
```
