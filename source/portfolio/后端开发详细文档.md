## 📡 通信协议

### WebSocket 消息格式

```typescript
// 客户端 -> 服务器
interface WSRequest {
  type: string;           // 消息类型
  data: any;              // 消息数据
  seq?: number;           // 序列号(可选)
}

// 服务器 -> 客户端
interface WSResponse {
  type: string;           // 消息类型
  code: number;           // 状态码 0=成功
  data: any;              // 响应数据
  msg?: string;           // 错误信息
}
```

### 消息类型定义

| 消息类型 | 方向 | 说明 |
|---------|------|------|
| `heartbeat` | 双向 | 心跳保活 |
| `login` | C→S | 用户登录 |
| `login_result` | S→C | 登录结果 |
| `create_room` | C→S | 创建房间 |
| `join_room` | C→S | 加入房间 |
| `room_info` | S→C | 房间信息 |
| `player_join` | S→C | 玩家加入 |
| `player_leave` | S→C | 玩家离开 |
| `game_start` | S→C | 游戏开始 |
| `action` | C→S | 玩家操作 |
| `action_notify` | S→C | 操作通知 |
| `game_over` | S→C | 游戏结束 |
| `settle` | S→C | 结算 |

### 心跳机制
```typescript
// 客户端: 每30秒发送一次心跳
{ type: 'heartbeat', data: { timestamp: Date.now() } }

// 服务器: 超过60秒未收到心跳则断开连接
```

### 错误码定义

| 错误码 | 说明 |
|--------|------|
| 0 | 成功 |
| 1001 | 参数错误 |
| 1002 | 令牌无效/过期 |
| 1003 | 权限不足 |
| 2001 | 用户不存在 |
| 2002 | 金币不足 |
| 3001 | 房间不存在 |
| 3002 | 房间已满 |
| 3003 | 房间已开始游戏 |
| 3004 | 房主不能离开 |
| 4001 | 游戏状态错误 |
| 4002 | 不是你的回合 |
| 4003 | 操作无效 |

### 断线重连
```typescript
// 客户端断线后重新连接
{ 
  type: 'reconnect', 
  data: { 
    userId: "用户ID",
    token: "登录令牌"
  } 
}
```

## 🗄️ 数据库设计 (MySQL + Redis)

### MySQL - 持久化数据

```sql
-- 用户表
CREATE TABLE users (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  openid VARCHAR(64) NOT NULL UNIQUE,  -- 微信openid
  nickname VARCHAR(64),
  avatar VARCHAR(255),
  gold INT DEFAULT 1000,              -- 金币
  diamond INT DEFAULT 0,              -- 钻石
  vip_level INT DEFAULT 0,            -- VIP等级
  create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  update_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_openid(openid)
);

-- 牌局记录表
CREATE TABLE game_records (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  room_id VARCHAR(32) NOT NULL,
  room_type VARCHAR(16) NOT NULL,      -- 房间类型
  user_ids JSON,                       -- 玩家ID列表
  winners JSON,                        -- 赢家ID列表
  total_pot INT,                      -- 总奖池
  create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_room_id(room_id),
  INDEX idx_create_time(create_time)
);

-- 用户游戏记录表
CREATE TABLE user_game_stats (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT NOT NULL,
  game_count INT DEFAULT 0,           -- 总游戏局数
  win_count INT DEFAULT 0,            -- 获胜局数
  max_win INT DEFAULT 0,              -- 最大赢取
  total_gain INT DEFAULT 0,           -- 总收益
  update_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_user_id(user_id)
);

-- 金币流水表
CREATE TABLE gold_logs (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT NOT NULL,
  amount INT NOT NULL,                -- 变动数量(正/负)
  reason VARCHAR(32) NOT NULL,         -- 变动原因
  balance INT NOT NULL,               -- 变动后余额
  create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_user_id(user_id),
  INDEX idx_create_time(create_time)
);
```

### Redis - 实时数据

#### 1. 用户会话
```redis
# String: 用户会话信息
session:{userId} -> {
  token: "jwt_token",
  socketId: "websocket_id",
  roomId: "当前所在房间ID"
}

# String: 登录令牌
token:{token} -> userId
```

#### 2. 房间信息 (实时)
```redis
# Hash: 房间数据
room:{roomId} -> {
  roomId: "房间ID",
  roomType: "房间类型",
  maxPlayers: 4,
  currentPlayers: 2,
  status: "waiting|playing|closed",
  ownerId: "房主ID",
  bet: 100,
  createTime: 1234567890
}

# Set: 房间内的玩家
room:{roomId}:players -> [userId1, userId2, ...]

# List: 房间列表
rooms:list -> [roomId1, roomId2, ...]
```

#### 3. 游戏状态 (实时)
```redis
# Hash: 游戏进行中的状态
game:{roomId} -> {
  currentRound: 1,
  totalRounds: 10,
  currentPlayerId: "当前行动玩家ID",
  status: "betting|playing|settling",
  pot: 400,
  createTime: 1234567890
}

# List: 游戏操作历史
game:{roomId}:history -> [action1, action2, ...]
```

#### 4. 玩家房间状态 (实时)
```redis
# Hash: 玩家房间状态
player:{userId}:room -> {
  roomId: "房间ID",
  seatIndex: 0,
  gold: 800,
  status: "ready|playing|folded|left",
  lastAction: "bet|call|raise|fold"
}
```

#### 5. 排行榜 (缓存)
```redis
# Sorted Set: 金币排行榜
rank:gold -> {
  userId1: 10000,
  userId2: 8000
}

# String: 排行榜最后更新时间
rank:gold:update_time -> 1234567890
```

### 数据同步策略

| 数据类型 | 存储 | 说明 |
|---------|------|------|
| 用户信息 | MySQL | 持久化存储 |
| 金币余额 | MySQL + Redis | 双写，Redis优先 |
| 牌局记录 | MySQL | 持久化存储 |
| 游戏状态 | Redis | 实时内存 |
| 房间状态 | Redis | 实时内存 |
| 排行榜 | Redis | 缓存，定期同步MySQL |

## 🎮 核心功能模块

### 1. 登录模块

**流程:**
```
用户打开小程序
    ↓
获取微信授权 code
    ↓
发送给后端验证
    ↓
返回用户信息 + JWT token
    ↓
保存到本地存储
```

**接口:**
- `POST /api/login` - 微信登录
- `POST /api/refresh_token` - 刷新令牌
- `GET /api/user/info` - 获取用户信息
- `POST /api/user/update` - 更新用户信息

**请求/响应示例:**
```typescript
// POST /api/login
// Request:
{
  code: "微信授权code"
}

// Response:
{
  code: 0,
  data: {
    userId: 123,
    nickname: "玩家昵称",
    avatar: "头像URL",
    gold: 1000,
    token: "jwt_token"
  }
}
```

### 2. 房间管理

**功能:**
- 创建房间 (房主)
- 加入房间 (普通玩家)
- 退出房间
- 房主解散房间
- 踢人 (房主)
- 房间设置 (底注/局数)

**状态:**
- `waiting` - 等待玩家
- `ready` - 玩家准备
- `playing` -游戏中
- `closed` - 已关闭

### 3. 游戏逻辑

**流程:**
```
发牌阶段
    ↓
庄家选择(如需要)
    ↓
行动阶段(循环)
    → 玩家操作(bet/call/raise/fold)
    → 检查是否结束
    ↓
结算阶段
    → 计算输赢
    → 更新金币
    → 排行榜更新
```

**操作类型:**
- `bet` - 下注
- `call` - 跟注
- `raise` - 加注
- `fold` - 弃牌
- `check` - 看牌
- `allin` - 全下

### 4. 结算系统

**计算:**
```typescript
// 获胜判断
winner = compareHands(player1Cards, player2Cards)

// 输赢计算
winAmount = pot * (winCount / totalPlayers)

// 金币更新
player.gold += winAmount

// 排行榜更新
zadd('rank:gold', player.gold, player.userId)
```

### 5. 战绩查询

**数据:**
- 总局数
- 胜率
- 最大赢取
- 当前排名

## 🔒 安全考虑

### 1. 身份验证
- JWT Token 验证
- Token 过期时间设置
- 敏感操作需要重新验证

### 2. 数据安全
- HTTPS 加密传输
- 数据库 SQL 注入防护
- XSS 攻击防护

### 3. 游戏安全
- 关键逻辑服务端验证
- 禁止客户端发牌/结算
- 频率限制(防刷)

### 4. 金钱系统
- 金币变化需要记录流水
- 服务端校验金币余额
- 敏感操作日志记录
