计算机网络基础

网络基础知识是每个开发者都应该掌握的核心技术。本文将详细介绍 TCP/IP 协议栈、DNS 域名系统和 HTTP 协议等网络基础知识。

一、网络协议概述

网络协议是计算机网络中进行数据交换而建立的规则、标准或约定的集合。它规定了通信双方如何建立连接、如何传输数据以及如何处理错误等。

常见的网络协议

协议 层次 说明
HTTP 应用层 Web 数据传输
DNS 应用层 域名解析
TCP 传输层 可靠传输
IP 网络层 路由转发
Ethernet 数据链路层 局域网传输

二、TCP/IP 协议栈

TCP/IP 是互联网的基础协议族，它是一组协议的集合，包括 TCP、IP、UDP、HTTP 等多个协议。

TCP/IP 四层模型

各层功能说明

应用层

  • 为用户提供网络应用服务
  • 直接与用户交互
  • 协议：HTTP、HTTPS、FTP、DNS、SMTP 等

传输层

  • 提供端到端的可靠传输
  • 负责流量控制、差错控制
  • 协议：TCP（可靠）、UDP（快速）

网络层

  • 实现主机到主机的通信
  • 负责路由和寻址
  • 协议：IP、ICMP、ARP

网络接口层

  • 负责物理设备之间的数据传输
  • 协议：Ethernet、PPP

三、IP 地址

IP 地址是分配给网络设备的唯一标识符，用于在网络中定位设备。

IPv4 地址

IPv4 地址分类：

类别 范围 用途
A 类 1.0.0.0 - 126.255.255.255 大型网络
B 类 128.0.0.0 - 191.255.255.255 中型网络

  • 10.x.x.x：私有地址（局域网）
  • 172.16.x.x - 172.31.x.x：私有地址（局域网）

四、TCP 协议

TCP（Transmission Control Protocol）是面向连接的可靠传输协议。

TCP 特点

  • 面向连接：建立连接后传输数据
  • 可靠传输：确认机制、重传机制
  • 全双工：可同时收发数据
  • 字节流：无消息边界

TCP 三次握手

建立 TCP 连接需要三次握手：

三次握手过程：

  1. 第一次握手：客户端发送 SYN 包（SYN=1，seq=x），进入 SYN_SENT 状态
  2. 第二次握手：服务器收到 SYN 包，发送 SYN+ACK 包（SYN=1，ACK=1，seq=y，ack=x+1），进入 SYN_RCVD 状态
  3. 第三次握手：客户端收到 SYN+ACK 包，发送 ACK 包（ACK=1，seq=x+1，ack=y+1），进入 ESTABLISHED 状态

TCP 状态转换

常见状态：

  • LISTEN：监听连接
  • SYN_SENT：已发送 SYN
  • SYN_RCVD：已收到 SYN
  • ESTABLISHED：连接已建立
  • FIN_WAIT_1：已发送 FIN
  • FIN_WAIT_2：已收到对方 ACK
  • CLOSING：双方同时关闭
  • TIME_WAIT：等待最后确认
  • CLOSED：连接已关闭

TCP 窗口与流量控制

TCP 使用滑动窗口机制进行流量控制：

  • 拥塞控制：防止网络过载
  • 慢启动：从小窗口开始
  • 拥塞避免：线性增长
  • 快速重传：收到 3 个重复 ACK

五、UDP 协议

UDP（User Datagram Protocol）是面向无连接的不可靠传输协议。

UDP 特点

  • 无连接：不需要建立连接
  • 不可靠：不保证送达
  • 高效：头部开销小（8 字节）
  • 无流量控制：适合实时应用

UDP 应用场景

  • DNS 查询
  • 视频流
  • 语音通话
  • 在线游戏
  • DHCP

TCP vs UDP

特性 TCP UDP
连接性 面向连接 无连接
可靠性 可靠 不可靠
头部 20-60 字节 8 字节
速度 较慢 较快
适用 文件、邮件 实时、音视频

六、DNS 域名系统

域名结构

域名级别

  • 顶级域名：.com、.org、.edu、.cn、.net 等
  • 二级域名：example、google、baidu 等
  • 子域名：www、mail、blog 等

方法 说明 幂等

1xx - 信息响应

  • 100 Continue
  • 101 Switching Protocols

2xx - 成功响应

  • 200 OK
  • 201 Created
  • 204 No Content

3xx - 重定向

  • 301 Moved Permanently
  • 302 Found
  • 304 Not Modified

4xx - 客户端错误

  • 400 Bad Request
  • 401 Unauthorized
  • 403 Forbidden
  • 404 Not Found
  • 405 Method Not Allowed

5xx - 服务器错误

  • 500 Internal Server Error
  • 502 Bad Gateway
  • 503 Service Unavailable
  • 504 Gateway Timeout

请求头 说明

响应头 说明

HTTP 版本

HTTP/1.0

  • 每次请求建立新连接
  • 不支持持久连接
  • 支持少量方法

HTTP/1.1

  • 支持持久连接
  • 支持管道化
  • 支持分块传输
  • 增加更多方法

HTTP/2

  • 多路复用（并行传输）
  • Header 压缩
  • 服务器推送
  • 二进制分帧

HTTP/3

  • 基于 QUIC 协议
  • 0-RTT 连接
  • 更好的移动端体验
  • 解决队头阻塞

HTTPS

HTTPS = HTTP + TLS/SSL，提供加密传输。
